## 3. MVP 1차 범위

초기 MVP는 다음 기능에 집중한다.

- F-01 개인별 은퇴 목표 및 현재 재정 상태 입력
- F-02 건강보험료 예상 시뮬레이션
- F-09 은퇴 후 월별 현금흐름 시뮬레이션
- F-10 은퇴 자산 고갈 시점 예측
- F-11 은퇴 계획 진행 상황 대시보드

2차 이후 기능은 ISA, 연금, ETF, 투자 성향, 전문가 상담, 최신 정보, 체크리스트 기능이다.

## 5. 답변 방식

사용자가 질문하면 다음 순서로 답한다.

1. 요청이 문서, 이슈, 코드, 배포 중 어디에 해당하는지 파악한다.
2. 관련 기준 문서를 안내한다.
3. 목적 → 지금 할 일 → 하지 말 것 → 다음 단계 순서로 설명한다.
4. 기능 구현 요청이면 해당 F-xx의 완료 기준을 따른다.
5. 진행 상태는 GitHub Issues를 기준으로 판단한다.

## 6. 코드 작성 원칙

- React + TypeScript 기준으로 작성한다.
- 계산 로직은 화면 컴포넌트와 분리한다.
- 금액 계산은 내부적으로 원 단위를 사용한다.
- 화면 표시는 만원 단위를 기본으로 한다.
- 월별 현금흐름 계산은 월 단위를 기본으로 한다.
- 숫자 입력값은 검증한다.
- 음수 입력은 원칙적으로 차단한다.
- 결과 화면에는 기준일과 계산 가정을 표시한다.

## 8. GitHub Issues 운영 원칙

- 작업은 GitHub Issue 단위로 나눈다.
- 기능 Issue 제목은 `[F-xx] 작업명` 형식을 사용한다.
- 공통 Issue 제목은 `[공통] 작업명` 형식을 사용한다.
- Issue 본문에는 배경, 요구사항, 완료 기준을 포함한다.
- 완료 기준이 불명확하면 먼저 문서를 정리한다.
