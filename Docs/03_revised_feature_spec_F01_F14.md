# F-01~F-14 보완 기능 정의서

## 문서 목적

이 문서는 기존 F-01~F-14 기능 문서를 심층 분석 결과에 맞춰 다시 정리한 보완본입니다. 핵심 방향은 다음과 같습니다.

1. MVP 1차 기능과 고도화 기능을 분리한다.
2. 세금, 건강보험료, 투자 관련 기능에는 고지문과 기준일을 포함한다.
3. F-07과 F-08의 역할 중복을 제거한다.
4. F-01 입력 데이터를 모든 계산 기능의 기준 데이터로 사용한다.

---

## F-01 개인별 은퇴 목표 및 현재 재정 상태 입력

### 우선순위

P0 / MVP 1차

### 기능 목적

사용자의 은퇴 목표, 소득, 자산, 부채, 지출 정보를 입력받아 전체 서비스의 기준 데이터를 만든다.

### MVP 포함 항목

- 출생연도 또는 현재 나이
- 퇴직 예정 연도/월
- 은퇴 후 월 생활비
- 국민연금 예상 수령액 및 수령 시작 시점
- 퇴직연금, 개인연금, 금융자산, 부동산 자산
- 부채
- 월 고정지출 및 변동지출

### 완료 기준

- 필수값 누락 시 다음 단계로 이동하지 않는다.
- 금액 입력값은 숫자만 허용한다.
- 음수 입력은 차단한다.
- 입력 완료 후 F-02, F-09, F-10, F-11에서 재사용 가능하다.

---

## F-02 건강보험료 예상 시뮬레이션

### 우선순위

P1 / MVP 1차

### 기능 목적

은퇴 후 예상 건강보험료를 추정하고, 임의계속가입 또는 피부양자 가능성을 안내한다.

### MVP 포함 항목

- 지역가입자 건강보험료 추정
- 장기요양보험료 포함 여부 표시
- 임의계속가입 제도 안내
- 피부양자 가능성 체크 항목 안내

### 고지문

건강보험료는 국민건강보험공단 기준 및 개인별 소득·재산 상황에 따라 달라질 수 있으며, 본 결과는 추정치입니다.

### 완료 기준

- 계산 기준일이 표시된다.
- 보험료 추정치와 안내 문구가 함께 표시된다.

---

## F-03 ISA 계좌 개설 및 운용 가이드

### 우선순위

P2 / 2차 기능

### 기능 목적

ISA 계좌의 개념, 개설 절차, 운용 방법, 주의사항을 안내한다.

### MVP 이후 포함 항목

- ISA 계좌 개념과 종류
- 가입 대상 및 개설 절차
- 납입 한도와 의무 보유 기간
- 중도 해지 시 유의사항
- 은퇴 준비 관점의 활용 방법

### 완료 기준

- 사용자가 ISA 계좌를 왜 활용해야 하는지 이해할 수 있다.
- 기준일과 주의사항이 표시된다.

---

## F-04 ISA 계좌를 통한 종합소득세 절세 효과 분석

### 우선순위

P2 / 2차 기능

### 기능 목적

ISA 계좌 활용 전후의 세금 부담 차이를 추정한다.

### 포함 항목

- ISA 활용 전/후 비교
- 금융소득 종합과세 기준 안내
- 비과세/분리과세 효과 설명
- 건강보험료 절감 가능성과의 연결 설명

### 고지문

세금 계산은 사용자가 입력한 값과 현재 문서 기준의 단순 가정에 따른 추정 결과입니다. 실제 세액은 국세청 또는 세무 전문가 확인이 필요합니다.

---

## F-05 퇴직연금 현황 분석 및 최적화 전략

### 우선순위

P2 / 2차 기능

### 기능 목적

DB, DC, IRP 현황을 입력받아 은퇴 소득 관점에서 요약 분석한다.

### 포함 항목

- 연금 유형별 입력
- 적립금, 수익률, 운용 상품 구성
- 일시금/연금 수령 비교 안내
- IRP 추가 납입과 세액공제 안내

### 주의사항

특정 상품 매수·매도 권유가 아니라 정보 제공과 시뮬레이션 목적임을 표시한다.

---

## F-06 개인연금 포트폴리오 진단 및 개선 방안

### 우선순위

P2 / 2차 기능

### 기능 목적

연금저축, 연금보험 등 개인연금 현황을 입력받아 수수료, 수익률, 수령 계획을 점검한다.

### 포함 항목

- 개인연금 유형
- 납입액, 수익률, 현재 잔액
- 수령 시작 시점
- 수수료와 세제 혜택 안내

### 완료 기준

- 사용자가 자신의 개인연금 상태를 요약해서 볼 수 있다.
- 개선 방향은 일반 가이드 수준으로 제공한다.

---

## F-07 은퇴 자산 운용을 위한 ETF 포트폴리오 구성

### 우선순위

P2 / 2차 기능

### 기능 목적

F-08 투자 성향 결과를 받아 자산군 중심의 포트폴리오 예시를 제공한다.

### 역할 분리

- F-08: 투자 성향 진단
- F-07: 진단 결과 기반 포트폴리오 구성

### 포함 항목

- 안정형/중립형/성장형 자산군 배분 예시
- 국내/해외, 주식/채권 등 분산 설명
- 리밸런싱 필요성 안내
- 투자 위험 고지

### 주의사항

초기에는 개별 ETF 종목 추천보다 자산군 배분 예시 중심으로 제공한다.

---

## F-08 투자 성향 분석

### 우선순위

P2 / 2차 기능

### 기능 목적

설문을 통해 사용자의 투자 성향을 분류한다.

### 포함 항목

- 투자 경험
- 손실 감내 수준
- 투자 기간
- 기대 수익률
- 안정성 선호도

### 결과 유형

- 보수형
- 안정추구형
- 위험중립형
- 적극투자형
- 공격투자형

### 완료 기준

- 설문 완료 후 성향 결과가 표시된다.
- 결과는 F-07 포트폴리오 구성 입력값으로 전달된다.

---

## F-09 은퇴 후 월별 현금흐름 시뮬레이션

### 우선순위

P0 / MVP 1차

### 기능 목적

은퇴 후 월별 수입, 지출, 순현금흐름, 누적 자산 변화를 계산한다.

### MVP 포함 항목

- 국민연금, 퇴직연금, 개인연금, 기타소득
- 생활비, 의료비, 건강보험료, 재산세, 보험료
- 월별 수입/지출/순현금흐름
- 누적 자산 잔액
- 부족 현금 발생 월 표시

### 완료 기준

- 월별 결과가 표 또는 그래프로 표시된다.
- 사용자가 가정값을 수정할 수 있다.

---

## F-10 은퇴 자산 고갈 시점 예측

### 우선순위

P1 / MVP 1차

### 기능 목적

F-09 결과를 바탕으로 자산이 고갈될 수 있는 시점을 예측한다.

### MVP 포함 항목

- 고갈 예상 연도/월
- 고갈 예상 나이
- 생활비 증가 시나리오
- 수익률 하락 시나리오
- 의료비 증가 시나리오

### 완료 기준

- 기본 시나리오와 최소 2개 대안 시나리오가 표시된다.
- 고갈 위험이 있는 경우 대응책이 함께 표시된다.

---

## F-11 은퇴 계획 진행 상황 대시보드

### 우선순위

P1 / MVP 1차

### 기능 목적

F-01, F-02, F-09, F-10 결과를 한 화면에서 요약한다.

### MVP 포함 항목

- 월 수입
- 월 지출
- 월 순현금흐름
- 예상 건강보험료
- 예상 자산 고갈 시점
- 은퇴 준비 상태 메시지

### 완료 기준

- 입력 데이터가 부족하면 필요한 다음 입력을 안내한다.
- 계산 결과가 있으면 핵심 지표가 카드 형태로 표시된다.

---

## F-12 전문가 상담 연결

### 우선순위

P3 / 고도화

### 기능 목적

복잡한 세무, 투자, 법률, 심리 상담이 필요한 사용자를 전문가 상담으로 연결한다.

### 초기 구현 권장

- 실제 전문가 자동 매칭이 아니라 상담 신청 폼으로 시작
- 상담 목적, 연락처, 희망 방식 입력
- 개인정보 처리 동의 필수

### 완료 기준

- 상담 신청이 저장된다.
- 관리자 또는 운영자가 확인 가능하다.

---

## F-13 최신 세법 및 금융 상품 정보 제공

### 우선순위

P3 / 고도화

### 기능 목적

은퇴 준비에 필요한 최신 정보를 제공한다.

### 초기 구현 권장

- 실시간 자동 업데이트가 아니라 수동 게시판으로 시작
- 각 게시글에 출처, 기준일, 업데이트일 표시
- 공식 기관 링크 중심으로 운영

### 완료 기준

- 최신 정보 목록과 상세 페이지가 있다.
- 각 정보에 출처와 기준일이 표시된다.

---

## F-14 은퇴 준비 체크리스트 및 목표 달성 알림

### 우선순위

P3 / 고도화

### 기능 목적

은퇴 준비 항목을 체크리스트로 제공하고 주요 일정에 알림을 제공한다.

### 초기 구현 권장

- 알림 기능 전에는 체크리스트 수동 완료 기능부터 구현
- 필수/선택 항목 구분
- 진행률 표시

### 완료 기준

- 사용자가 체크리스트를 완료 처리할 수 있다.
- 전체 진행률이 표시된다.
- 알림 기능은 사용자 동의 후 활성화한다.

---

## 공통 완료 조건

모든 기능은 다음 조건을 만족해야 합니다.

1. 입력값 검증이 있다.
2. 계산 기준일이 표시된다.
3. 결과가 추정치인 경우 고지문이 표시된다.
4. 모바일 화면에서도 사용 가능해야 한다.
5. 사용자가 다음에 무엇을 해야 하는지 알 수 있어야 한다.
