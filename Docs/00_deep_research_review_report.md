# 은퇴현금 서비스 문서 심층 분석 보고서

## 1. 검토 대상

- README.md: 은퇴현금 서비스 기획서
- F-01 ~ F-14 주요 기능 문서
- GitHub Issues 등록 과정에서 도출된 공통 리스크

## 2. 종합 판단

현재 문서는 은퇴 준비 전반을 폭넓게 다루고 있어 서비스 방향성은 좋지만, 초기 MVP로 구현하기에는 범위가 과도하게 넓습니다. 특히 건강보험료, 세금, ISA, 퇴직연금, 개인연금, ETF, 자산 고갈 예측, 전문가 상담, 최신 정보 제공, 알림 기능까지 한 번에 포함되어 있어 개발 난이도와 운영 부담이 큽니다.

따라서 1차 MVP는 사용자의 핵심 문제인 "은퇴 후 매달 현금흐름이 괜찮은가"를 검증하는 방향으로 축소하는 것이 적절합니다.

## 3. 핵심 이슈 요약

### 3.1 MVP 범위 과대

README와 F-01~F-14 문서는 종합 은퇴 플랫폼 수준의 기능을 포함합니다. 초기 서비스에서는 모든 기능을 구현하기보다 현금흐름 설계 중심으로 좁히는 것이 좋습니다.

권장 MVP 1차 범위:

- F-01 개인별 은퇴 목표 및 현재 재정 상태 입력
- F-02 건강보험료 예상 시뮬레이션
- F-09 은퇴 후 월별 현금흐름 시뮬레이션
- F-10 은퇴 자산 고갈 시점 예측
- F-11 은퇴 계획 진행 상황 대시보드

2차 이후 확장 범위:

- F-03 ISA 계좌 개설 및 운용 가이드
- F-04 ISA 절세 효과 분석
- F-05 퇴직연금 분석
- F-06 개인연금 분석
- F-07 ETF 포트폴리오 구성
- F-08 투자 성향 분석
- F-12 전문가 상담 연결
- F-13 최신 세법 및 금융 상품 정보
- F-14 체크리스트 및 알림

### 3.2 법률·세무·투자 고지문 필요

서비스는 건강보험료, 종합소득세, 연금, ETF, 포트폴리오 개선 등을 다루므로 사용자가 세무 자문 또는 투자 자문으로 오해할 수 있습니다.

필수 고지:

- 본 서비스는 정보 제공 및 추정 시뮬레이션 목적입니다.
- 실제 세금, 건강보험료, 투자 결과는 개인 상황과 제도 변경에 따라 달라질 수 있습니다.
- 최종 의사결정 전 국민건강보험공단, 국세청, 금융기관 또는 전문가 확인이 필요합니다.

### 3.3 F-07과 F-08 기능 중복

F-07은 ETF 추천 및 포트폴리오 구성, F-08은 투자 성향 분석 기반 ETF 전략 수립으로 정의되어 있어 기능 중복 가능성이 큽니다.

역할 분리 권장:

- F-08: 투자 성향 설문 및 결과 산출 전용
- F-07: F-08 결과를 받아 ETF/자산군 포트폴리오 구성 전용

### 3.4 데이터 모델 미정의

F-01에서 입력한 데이터가 F-02, F-04, F-05, F-06, F-09, F-10, F-11에 모두 사용됩니다. 하지만 현재 문서에는 공통 데이터 모델, 금액 단위, 월/연 기준, 필수값 정의가 부족합니다.

필요한 공통 타입:

- UserProfile
- RetirementGoal
- Income
- Expense
- Asset
- Liability
- Pension
- ScenarioAssumption
- CashflowResult

### 3.5 최신 정보 제공 기능의 운영 부담

F-13은 최신 세법, 금융 시장, 상품 정보를 실시간으로 제공한다고 되어 있으나, 초기 서비스에서 실시간 정보 수집과 검증은 부담이 큽니다.

권장 방향:

- MVP: 수동 업데이트 가능한 정보 게시판
- 2차: 뉴스레터/알림
- 3차: 외부 API 또는 크롤링 기반 자동 업데이트

## 4. 우선순위 제안

| 우선순위 | 기능/이슈 | 이유 |
|---|---|---|
| P0 | 공통 데이터 모델 정의 | 모든 계산 기능의 기반 |
| P0 | 세금·건보료·투자 고지문 | 신뢰와 책임 범위 명확화 |
| P0 | F-01 | 모든 기능의 입력 기반 |
| P0 | F-09 | 서비스 핵심 가치인 현금흐름 확인 |
| P1 | F-02 | 은퇴 후 고정지출 핵심 요소 |
| P1 | F-10 | 자산 지속가능성 판단 |
| P1 | F-11 | 결과를 한눈에 보여주는 대시보드 |
| P2 | F-03~F-06 | 연금·절세 정보 확장 |
| P2 | F-07~F-08 | 투자 기능은 고지와 범위 통제 필요 |
| P3 | F-12~F-14 | 운영/알림/상담 고도화 영역 |

## 5. 권장 실행 순서

1. README에 MVP 1차 범위와 고도화 범위를 분리한다.
2. Docs에 공통 데이터 모델 문서를 추가한다.
3. F-01, F-09, F-10 중심의 계산 흐름을 먼저 만든다.
4. F-02 건강보험료는 추정치와 기준일을 명확히 표시한다.
5. F-11 대시보드는 입력 부족 상태와 결과 상태를 나누어 설계한다.
6. F-03~F-14는 2차 이후 기능으로 관리한다.

## 6. 결론

현재 문서의 방향성은 좋지만, 초기 개발에서는 "은퇴 후 현금흐름을 월별로 확인하고 자산 고갈 위험을 예측한다"는 핵심 가치에 집중해야 합니다. 이를 위해 기능을 단계별로 나누고, 데이터 모델과 고지문을 먼저 정리한 뒤 개발을 진행하는 것이 안전합니다.
